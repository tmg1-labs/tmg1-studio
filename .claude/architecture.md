# アーキテクチャ方針

## 全体構成

TMG1 Studio は、動画を **1bit モノクロ（`monob`）に区間ごとに設定を変えて**変換する
デスクトップ GUI。単一の一様なモノクロ設定では暗部にノイズが出たり細部が潰れたりするため、
タイムラインを区間（segment）に分割し、区間ごとにコントラスト・レベル絞り・ディザを
個別調整してプレビューしながら仕上げる。

```
tmg1-studio/
├── src-tauri/            ... Rust バックエンド (Tauri v2)
│   ├── src/filter.rs         ... フィルタチェーンビルダ（唯一の真実の源 / single source of truth）
│   ├── src/ffmpeg.rs         ... ffmpeg / ffprobe プロセスラッパ
│   ├── src/lib.rs            ... Tauri コマンド (probe_video / render_preview / render_range / export)
│   └── src/main.rs           ... エントリポイント（lib.rs の run() を呼ぶだけ）
└── src/                  ... Web フロント (Vanilla TS): タイムライン / パラメータ / プレビュー
    ├── main.ts               ... UI ロジック全体
    └── styles.css
```

## 処理モデル

- **フィルタチェーンの単一ソース**: プレビューとエクスポートは同一の `filter.rs`
  でフィルタチェーンを組み立てる。**WYSIWYG（見たものが得られる）を保証する**ため、
  この二経路でフィルタ構築を分岐させない。
- 区間ごとのパラメータ:
  - **Contrast**（`eq=contrast`）
  - **Level squeeze** — 低しきい値未満の暗部を潰し、高しきい値超の明部を飛ばす
    （暗部の白点ノイズ・前景の抜けを除去）
  - **Dither** — Bayer / 誤差拡散 / なし（`-sws_dither`）
- **エクスポート**: 各区間を切り出し、それぞれの設定でトランスコードし、raw をロスレスに連結。
  あわせて目視確認用の最近傍アップスケール `.preview.mp4` を生成する。

## 技術選定の理由

- **Tauri v2**: クロスプラットフォームなネイティブ GUI を軽量に。ロジックは Rust、UI は Web。
- **Vanilla TS（フレームワークなし）**: 画面が単一で状態も限定的なため、React 等を持ち込まない。
- **ffmpeg を同梱せず PATH 参照**: バイナリ配布を軽くし、ユーザ環境の ffmpeg を尊重する。
- **プレビュー忠実度は `monob` まで**: TMG1 エンコードはロスレスなので、monob プレビュー＝
  実機ピクセルと一致する。TMG1 ラウンドトリップはプレビューに不要（`tmg1-cli` が担う）。

## 禁止パターン

- プレビュー用とエクスポート用でフィルタチェーン構築を分岐させない（`filter.rs` に一本化）。
- TMG1 / コーデックのアルゴリズムを本リポジトリに持ち込まない（責務は `tmg1-codec` / `tmg1-cli`）。
- ffmpeg バイナリを同梱しない（PATH 参照を維持）。
- 出力幅は 8 の倍数を前提にする（monob のバイト境界。破ると詰め方がずれる）。
