# Session Record スキル

現在のセッションで起きたことを `.claude/context/` 配下に記録するスキルです。
`code-project-init` が作った構成を前提に、**頻繁に変わる context だけ**を更新する。

## 対象ファイル（これ以外は触らない）

| ファイル | 書く内容 |
|---|---|
| `.claude/context/current-sprint.md` | 今やっていること・進捗・次にやること・一時的な制約 |
| `.claude/context/known-issues.md` | このセッションで判明したハマりポイント・地雷・回避策 |

`architecture.md` / `coding-style.md` / `workflows.md` / `CLAUDE.md` は**更新しない**。
それらに該当する変更に気づいた場合は、書き込まずに事後報告の中で「○○は workflows.md
向けの内容です」と指摘するだけに留める。

## 確定方法

**即時書込 → 事後報告**。承認を待たずに書き込み、完了後に変更点を要約して報告する。
（破壊的な全置換は避け、既存内容を尊重して追記・差し替えする）

## 手順

### Step 0: 前提を確認する

`.claude/context/` ディレクトリの存在を確認する。

- **存在しない場合** → 書き込まず、「このプロジェクトは未初期化です。`code-project-init`
  でセットアップしますか？」とユーザーに案内して終了する
- **存在する場合** → Step 1 へ

### Step 1: セッション内容を棚卸しする

このセッションの会話・操作から以下を抽出する（憶測で埋めない。なければ省く）。

- 完了した作業・変更したファイル
- 進行中で次セッションに持ち越す作業
- 新たに発生した一時的な制約・決定事項
- ハマった点・回避策・注意すべき地雷

### Step 2: current-sprint.md を更新する

1. 既存の `current-sprint.md` を読む
2. 完了した項目は「進行中」から外す（または完了として整理する）
3. 進行中・次にやることを最新の状態に書き換える
4. 日付が分かるものは絶対日付（YYYY-MM-DD）で書く

### Step 3: known-issues.md を更新する

1. 既存の `known-issues.md` を読む
2. このセッションで判明した問題・地雷・回避策を**追記**する（既存項目は消さない）
3. 既に解決した既知の問題があれば、解決済みとして整理する

### Step 4: 事後報告する

書き込み後、以下を簡潔に報告する。

- どのファイルに何を追記・変更したか
- 対象外（architecture/coding-style/workflows/CLAUDE.md）に該当しそうな内容があれば、
  その旨と推奨先ファイルを指摘

## 注意

- 機微情報（トークン・パスワード等）は記録しない
- 既存記録の大量削除はしない。整理が必要なら報告で提案する
